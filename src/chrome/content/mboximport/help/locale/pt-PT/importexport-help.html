Visão geral e menus do ImportExportTools NG

ImportExportTools NG é uma extensão complementar baseada no trabalho original de Paolo "Kaosmos". A extensão oferece múltiplas funções de importação e exportação, suportando diversos formatos de arquivo, incluindo: Observação: Alguns formatos não são suportados em todas as funções.
- mbox
- EML
- diretório de e-mails
- HTML
- texto simples
- CSV (Valores Separados por Vírgula)
- PDF

Embora a maior parte da funcionalidade gire em torno de mensagens, também há suporte para importação e exportação de perfis do Thunderbird. Além disso, as exportações podem incluir um arquivo de índice HTML ou CSV. O backup do perfil do Thunderbird é outro recurso disponível.

O acesso às funcionalidades do IETng é feito através de vários menus diferentes instalados pela extensão. As funções são sensíveis ao contexto. Menus são adicionados para:
- O contexto do painel de pastas (operação de pastas)
- Contexto da lista de mensagens (mensagens selecionadas)
- O menu Ferramentas (operações de perfil e backup)

As opções acessíveis pelo menu "Ferramentas" não estão presentes no menu "Hambúrguer"; elas estarão em uma versão futura. Essas funções também estão disponíveis no novo botão da barra de ferramentas. Cada menu e botão também possui um item de menu Ajuda e Opções para acessar a caixa de diálogo de opções.

Formato de data personalizado

A opção de data personalizada permite o controle total sobre a formatação da data usada para ambos os arquivos no índice. Tanto o formato padrão "Data" quanto o novo formato personalizado "Data (personalizada)" estão disponíveis nas listas suspensas de nomes de arquivo. A data personalizada também pode ser usada com o token ${date_custom} nos nomes de arquivo personalizados estendidos. O formato de data original (padrão) é:
AAAAMMDDHHmm: 202011060808

O formato de data personalizado utiliza sequências de porcentagem (%) que correspondem à funcionalidade semelhante à função strftime() do Unix. Ele é baseado em uma implementação em JavaScript da função strftime() , que formata a data e a hora.
Adaptação da função strftime() por TH Doan (https://thdoan.github.io/strftime/)

Descrições de sequências de conversão

Sequência Descrição
%a Nome abreviado do dia da semana.
%A Nome completo do dia da semana.
%b Nome abreviado do mês.
%B Nome completo do mês.
%c Formato de data e hora preferencial (UTC) para a localidade atual.
%C Número do século (ano/100) como um número inteiro de 2 dígitos.
%d Dia do mês em formato decimal (entre 01 e 31).

%F Formato de data ISO 8601 (equivalente a %Y-%m-%d ).

Este tem o mesmo formato e valor que %Y , exceto que, se o número da semana ISO pertencer ao ano anterior ou seguinte, esse ano será usado em vez do anterior.
%g Semelhante %G , mas sem o século, ou seja, com um ano de dois dígitos (00-99).
%H Hora como um número decimal usando um relógio de 24 horas (intervalo de 00 a 23). Veja também %k .
%I Hora como um número decimal usando um relógio de 12 horas (intervalo de 01 a 12). Veja também %l .

%M Minuto em formato decimal (intervalo de 00 a 59).
%p "AM" ou "PM", de acordo com o horário especificado. Meio-dia é considerado "PM" e meia-noite, "AM".
%P Semelhante %p , mas em minúsculas ("am" ou "pm").
%s Número de segundos desde a Época, 1970-01-01 00:00:00 +0000 (UTC).
%S Em segundo lugar, como um número decimal (intervalo de 00 a 59).
%u Dia da semana em formato decimal (intervalo de 1 a 7), sendo segunda-feira o dia 1. Veja também %w .

Notas de compatibilidade:

  • %c - a string formatada é ligeiramente diferente
  • %D - não implementado (use %m/%d/%y ou %d/%m/%y )
  • %e - espaço não é adicionado
  • %E - não implementado
  • %h - não implementado (use %b )
  • %k - espaço não adicionado
  • %n - semelhante %m , mas sem zero à esquerda (use \n para nova linha)
  • %O - não implementado
  • %r - não implementado (use %I:%M:%S %p )
    • ${subject} O assunto da mensagem
  • %R - não implementado (use %H:%M )
  • %t - não implementado (use \t )
  • %T - não implementado (use %H:%M:%S )
  • %U - não implementado
  • %W - não implementado
  • %+ - não implementado
  • %% - não implementado (use % )

Preferências novas e experimentais (opções)

Existem diversos novos recursos e opções que ainda não podem ser configurados na caixa de diálogo de opções. Assim que esses recursos estiverem estabilizados, serão adicionados à interface do usuário.
Para utilizá-las, elas devem ser configuradas no Editor de Configuração. Para acessar o editor, use o menu Ferramentas/Opções, selecione a seção Avançado e você encontrará um botão "Editor de Configuração" na parte inferior. Se você digitar "importexporttoolsng" na barra de pesquisa, encontrará todas as preferências da extensão. Em seguida, você poderá editar as diversas preferências abaixo, conforme necessário.

importexporttoolsng.export.index_date_custom_format

Essa preferência segue a mesma formatação completa a ser usada para os arquivos de índice. Por padrão, é uma string vazia; se contiver uma string de data/hora formatada, o formato da data no índice será alterado.

importexporttoolsng.export.attachments.filename_extended_format

importexporttoolsng.export.embedded_attachments.filename_extended_format

Ambas as preferências utilizam o mesmo tipo de formato de nomenclatura de arquivo estendido que o nome do arquivo de mensagens exportadas. A primeira é para anexos externos, a segunda é para anexos incorporados (em linha).

importexporttoolsng.experimental.use_delivery_date

Normalmente, a 'Data' de uma mensagem no Thunderbird é a data definida pelo cliente de e-mail do remetente. Definir esta opção como 'verdadeiro' utiliza a data de recebimento no cliente de e-mail do destinatário. Esta data será usada tanto para os nomes dos arquivos quanto para a data nos arquivos de índice.

Recurso básico e global de teclas de atalho.

importexporttoolsng.experimental.hot_keys

Esta nova preferência permite ao usuário configurar até 10 atalhos de teclado. Os atalhos podem executar a maioria das funções do ImportExportTools NG, bem como outras funções do Thunderbird. A string de preferência é formatada como um objeto JSON. O formato e as teclas são os seguintes:

Exemplos de entradas de teclas de atalho (matriz de objetos JSON)

id - o índice da chave (1-10)
key - a chave (caractere único)
keycode - o código de tecla definido 'VK_*'
Pode ser incluída a chave ou o código de ativação, mas não ambos.
modifiers - os modificadores principais, listados separados por espaço ('control', 'shift', 'alt', 'accel')
oncommand - uma string de comando direta
contexts - o contexto da janela onde a tecla está ativa
'all' - todas as janelas em Messenger, Compor e Pesquisar
'messenger' - a janela principal do Messenger e suas variantes
'compose' - a janela de composição de mensagens
'search' - a caixa de diálogo de pesquisa
